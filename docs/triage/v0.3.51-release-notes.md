## Release body (paste into `gh release create`)

### Fixed: strict V4L2 clients (OBS, some capture tools) dropping out on camera startup

The libcamera pipeline takes ~2–3 seconds to produce its first frame. During that
window the relay device went completely silent — and strict clients give up long
before that. OBS in particular uses a 166 ms `select()` timeout, so it would
disconnect before the camera ever came up.

The camera-relay monitor now keeps the device fed with black frames while the
pipeline warms up. Measured on the bench (`camera-relay/tests/relay-loop-bench.c`):
the longest silence a client sees during startup drops from **2506 ms → 102 ms**,
comfortably inside OBS's timeout. Streaming performance is unchanged — 30 fps
delivery is identical to before (33.33 ms mean, zero late frames).

**Affects all webcam users** (Book3/Book4/Book5). Re-run your `install.sh` to pick
it up.

### Fixed: install URLs 404'ing after the repo rename

The repo is `samsung-galaxy-book-linux-fixes`; several scripts and docs still
pointed at the old name, so copy-pasted install commands 404'd. All shipped
occurrences fixed. Thanks **@LucasDondo** (#58).

### Added: NixOS support for the Book5 webcam fix

A declarative NixOS module for the Book5 (IPU7 / OV02E10) camera stack, plus a
`relayColorFilter` option for applying GStreamer elements to the relay output.
Thanks **@ang3lo-azevedo** (#60).

### Fixed: "Broken pipe" (EPIPE) on newer IPU7 kernels — *NixOS path*

Newer IPU7 kernels enforce V4L2 link validation strictly, and the old bayer fix
changed the kernel-facing `videoFormat`, which those kernels now reject ("Broken
pipe"). Bayer fix **v0.7** keeps `videoFormat` matching the sensor's media bus
code and feeds the corrected flipped Bayer order to the SoftwareISP through a
separate `ispFormat` instead.

⚠️ **This currently reaches NixOS users only** — the `.patch` is consumed by the
Nix overlay, while Ubuntu/Fedora/Arch get the bayer fix through the shell
installer's patcher, which still carries the older logic. Porting it is tracked in
**#69**, which also needs an OV02E10 tester on a non-NixOS distro. If you're on
kernel 7.0.10+ and hitting "Broken pipe", that issue is the one to watch.

Confirmed working on **Galaxy Book5 Pro 16" (960XHA)**, NixOS, kernel
7.0.10-cachyos.

### Internal

- `camera-relay/tests/relay-loop-bench.c` — a hardware-free regression test for the
  relay frame loop. Exercises both properties that have bitten us (no 30 fps
  stutter; no client timeout on startup) against a synthetic producer, and exits
  non-zero on either regression. No Galaxy Book required to run it.
