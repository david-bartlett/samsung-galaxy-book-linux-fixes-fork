## Release body (paste into `gh release create`)

### Fixed: camera never worked on Fedora (Galaxy Book3 / Book4)

If you're on Fedora with a Book3/Book4, the webcam fix installed "successfully"
and then never worked — the browser wouldn't list the camera, though GNOME
Snapshot would.

The installer tried `dnf install -y v4l2loopback`, but **that package doesn't
exist in Fedora's repos** — v4l2loopback lives in RPM Fusion (free) as
`akmod-v4l2loopback`. The command failed, the error was swallowed, and the install
reported success anyway. Without the loopback module, camera-relay crash-loops
forever and there is no V4L2 device for a browser to find. (Snapshot worked
because it bypasses the relay and talks to libcamera through PipeWire directly.)

The installer now enables RPM Fusion, installs `akmod-v4l2loopback`, builds it
against the running kernel, and — importantly — **re-checks that the module is
actually available**, failing loudly instead of leaving a crash-looping relay
behind.

This had been fixed in the Book5 installer back in v0.3.42 (#49) but was never
ported to the Book3/Book4 one. Thanks **@mateusluizfb** (#51), whose logs found it.

**If you're an affected Fedora user, re-run `webcam-fix-libcamera/install.sh`.**

### Fixed: install URLs 404'ing after the repo rename

The repo is `samsung-galaxy-book-linux-fixes`; several scripts and docs still
pointed at the old name, so copy-pasted install commands 404'd. All shipped
occurrences fixed. Thanks **@LucasDondo** (#58).

### Internal

- `camera-relay/tests/relay-loop-bench.c` — a hardware-free regression test for the
  relay frame loop. Exercises both properties that have bitten us (no 30 fps
  stutter; no client timeout on startup) against a synthetic producer, and exits
  non-zero on either regression. No Galaxy Book required to run it.

**Full changelog:** https://github.com/Andycodeman/samsung-galaxy-book-linux-fixes/compare/v0.3.50...v0.3.51

  ⚠️ Reaches **NixOS only**; the shell-installer port is tracked in **#69**.
